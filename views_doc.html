Upload Tools Documentation¶

upload.file_ok(line_1)¶

check if the file is legit

to be modified, this function is not enough to check the integrity of the file given

Parameters:line_1 – first line of the csv file, only the header is checked
upload.get_type(string, radiometric)¶

Read the measurement type from the string read, if it doesn’t exist, create it

Parameters:
  • string – string containing the measurement type, i.e. in “Rho_wn_IS_412” the type is “Rho_wn_IS”
  • radiometric – boolean, true if the measurement is radiometric, meaning the string contains the wavelength
upload.get_units(type)¶

Associate a measurement type with its units, to be completed with new types

Parameters:type – measurement type (string)
upload.get_wavelength(string)¶

Read the measurement wavelength from the string, if it doesn’t exist, create it

Parameters:string – string containing the wavelength, i.e. in “Rho_wn_IS_412” the type is “412”
upload.read_campaign_name(filename)¶

Get the name of the campaign from the name of the file (this function may should be rewritten or completed)

Parameters:filename – name of the file
upload.read_data(file_data, instrument, filename)¶

Function reading the file and uploading the data to the database

Parameters:
  • file_data – file object to be read
  • instrument – id of the instrument used for the measurement
  • filename – name of the file, the name of the campaign is extracted form it
upload.read_deployment_data(data, campaign, instrument)¶

Read the deployment data and create a deployment object

Parameters:
  • data – array containing the data
  • campaign – campaign object to be linked with the deployment(s)
  • instrument – database id of the instrument to be linked with the measurements
upload.read_file(file_data)¶

Creates an array (actually a list of lists) based on the uploaded data

Parameters:file_data – file object to be read
upload.read_point_data(data, deployment, instrument, header)¶

Read the point data, create point objects, read the measurement data and create measurement objects linked to the point

Parameters:
  • data – line of the array we read
  • deployment – deployment object to be linked with the points
  • instrument – database id of the instrument to be linked with the measurements
  • header – first line of the array containing the measurement types
